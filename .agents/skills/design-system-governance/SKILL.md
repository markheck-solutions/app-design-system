# Design System Governance

Purpose: safely evolve this design system.

Steps:

1. Read `AGENTS.md`, `DESIGN_SYSTEM_VERSION.md`, and impacted docs.
2. Update docs/tokens/examples.
3. Add decision records when the change should persist across apps.
4. Update changelog/version where needed.
5. Validate JSON/tokens.
6. Avoid app-specific rules unless generalized.
7. Keep framework-agnostic unless explicitly approved.

Required closeout:

- Validation command and result.
- Files changed.
- Version/changelog decision.
- Any Figma link or MCP log update.
