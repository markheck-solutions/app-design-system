# App Facelift

Purpose: audit and facelift one app using this design system.

Steps:

1. Read `AGENTS.md` and required docs.
2. Inspect target app.
3. Identify stack/routes/UI library.
4. Capture baseline screenshots.
5. Classify app.
6. Select target layout pattern.
7. Use Figma target frame if available.
8. Produce plan before code changes.
9. Preserve business logic.
10. Implement in small changes.
11. Run visual QA.
12. Update adoption matrix.

Hard rules:

- Do not freestyle visual design.
- Do not add UI dependencies without approval.
- Do not change backend behavior or data contracts unless explicitly approved.
- Provide before/after screenshots or equivalent visual proof.
