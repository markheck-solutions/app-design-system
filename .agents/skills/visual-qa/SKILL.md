# Visual QA

Purpose: validate implementation against design system and Figma.

Include:

- Before/after screenshots.
- Target frame comparison.
- Responsive checks.
- Accessibility checks.
- Status/severity consistency.
- Forbidden visual pattern checks.
- Pass/fail report.

Minimum checks:

- Target Figma frame linked.
- Baseline screenshots captured.
- After screenshots captured.
- Desktop and responsive layouts checked.
- Primary action clarity checked.
- Empty/loading/error states checked.
- Deviations from Figma documented.
