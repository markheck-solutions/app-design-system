# Figma Workflow

Figma is the visual source of truth.

GitHub is the written/enforceable source of truth.

The master file should be named: MS App Design System.

Use Figma Full seat capabilities.

Use remote Figma MCP by default when available.

Use working pages before approved pages.

Do not overwrite approved frames without explicit approval.

Store frame links in `figma/FIGMA_LINKS.md` and `apps/registry.json`.

Do not store Figma credentials.

Do not assume Code Connect.

## Per-app Workflow

1. Capture current state.
2. Place current state in Figma.
3. Create target directions.
4. Select approved frame.
5. Implement from approved frame.
6. Visually compare implementation to frame.
