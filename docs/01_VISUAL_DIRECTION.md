# Visual Direction

The shared direction is quality and polish, not identical appearance.

The apps should feel like a cohesive GitHub portfolio, not cloned screens. Shared rules create trust: accessibility, token discipline, status semantics, readable density, strong hierarchy, and implementation guardrails. Individual apps create memorability: product story, visual lane, hero moment, and visual signature.

Each app needs a unique product story. Codex must translate that story into app-specific visual choices before target design work starts.

## Shared Portfolio Feel

- High signal-to-noise.
- Strong hierarchy.
- Accessibility.
- Readable density.
- Operational clarity.
- Polished enough for executive and portfolio review.
- Clear status and severity semantics.
- Token discipline across color, spacing, type, elevation, radius, and motion.

## App-Specific Identity

Each app must define:

- Creative brief.
- Visual lane.
- Hero moment.
- Visual signature.
- Motion or interaction signature where useful.
- App-specific proof that the design is not a cloned dashboard skin.

## Rejected Traits

- Generic AI dashboard slop.
- Random neon gradients.
- Fake glassmorphism.
- Decorative charts without decision value.
- One-size-fits-all dashboard layouts.
- Reusing the same visual treatment across all apps.
- Toy SaaS styling.
- Oversized decorative icons.
- Excessive whitespace that reduces operational density.
- Weak contrast.
- Inconsistent statuses.
