# Form Patterns

## Form Sectioning

Group related fields into titled sections. Keep section order aligned to user workflow, not database order.

## Labels

Every field needs a persistent label. Placeholders are examples, not labels.

## Helper Text

Use helper text for format, consequence, and decision support. Do not repeat the label.

## Validation

Show validation close to the field. Summarize blocking validation near submit when many fields are involved.

## Required Fields

Required fields must be clear before submit. Do not rely only on asterisks without explanation.

## Error States

Error text must state what failed and how to fix it.

## Submit/Cancel Placement

Primary submit belongs at the end of the form or in a sticky action bar for long forms. Cancel/destructive actions must be visually secondary and explicit.

## Wizard Behavior

Use wizards when step order matters. Validate each step before advancing. Include review step for high-risk submissions.

## Accessibility Rules

Labels, descriptions, and errors must be programmatically associated where the stack supports it. Keyboard flow must match visual order.
