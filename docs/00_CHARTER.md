# Charter

## Goal

Facelift 8 apps with a consistent visual language that improves operational clarity, confidence, density, and polish.

## Non-goals

- No production component library yet.
- No forced redesign of business workflows.
- No assumption that all apps share the same frontend stack.
- No assumption of Figma Code Connect.

## Operating Principles

- Consistency: repeated UI problems should use repeated solutions.
- Density: screens should carry operational signal without becoming crowded.
- Accessibility: readable, keyboard-friendly, semantic interfaces are required.
- Operational clarity: blockers, statuses, risks, and next actions must be obvious.
- Visual polish: enterprise-ready presentation matters, but decoration must serve workflow value.
- Implementation realism: guidance must survive different app stacks.
- No fake functionality: do not add controls, charts, data, or workflows that do not work.
