# Accessibility

## Keyboard Navigation

All interactive controls must be reachable by keyboard. Focus order must follow visual workflow order.

## Focus States

Focus states must be visible and not rely on color alone.

## Semantic Buttons/Links

Use buttons for actions and links for navigation. Do not make non-interactive elements clickable without semantics.

## Color Contrast

Text, icons, borders, and status treatments must meet usable contrast. Critical operational text must be especially readable.

## Form Labels

All form controls need persistent labels and associated error/helper text.

## Table Accessibility

Use table semantics where stack allows. Keep headers clear and preserve row/column relationships.

## Reduced Motion

Motion must respect reduced-motion preferences where supported.

## Icon Labels

Icon-only controls require accessible names and visible tooltips when meaning is not obvious.

## Screen Reader Considerations

Status, error, loading, and completion changes should be exposed as text and announced when appropriate.
