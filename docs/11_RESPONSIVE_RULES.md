# Responsive Rules

## Desktop-first Operational Layout

The primary target is desktop operational work. Use wide layouts for tables, queues, dashboards, and detail views.

## Tablet Considerations

Tablet layouts should preserve key actions and status visibility. Collapse secondary panels before hiding primary data.

## Mobile Fallback

Mobile must remain usable, but not every dense operational workflow needs identical density. Prioritize identity, status, owner, deadline/risk, and action.

## Sidebar Collapse

Collapse sidebars when they compete with the primary work area. Preserve current location and primary nav access.

## Wide Tables

Prefer prioritized columns plus horizontal scroll before converting dense tables into low-information cards.

## Detail Drawers

Desktop drawers should preserve table context. Mobile drawers should become full-screen sheets with clear close and back behavior.

## KPI Strips

KPI strips wrap by priority. Do not shrink metric text until it becomes unreadable.
