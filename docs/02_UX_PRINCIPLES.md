# UX Principles

- Expose blockers quickly.
- Separate status from action.
- Make risk obvious.
- Keep tables readable.
- Make primary actions obvious.
- Avoid decorative complexity.
- Optimize for service delivery and operational workflows.
- Do not hide important actions in ambiguous icon-only controls.
- Design empty, loading, and error states intentionally.

## Implementation Notes

Every facelift plan must identify the main user workflow, the primary decisions the page supports, and the status/risk signals that must remain visible while users work.
